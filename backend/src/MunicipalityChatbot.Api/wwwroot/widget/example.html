Municipality Chatbot Widget

This page demonstrates how to embed the Municipality Chatbot widget on your website.

Demo Page Content

Click the chat button in the bottom-right corner to try the widget!

Quick Start

Add the following code to your website, just before the closing </body> tag:

<!-- Municipality Chatbot Widget -->
<script>
  window.MunicipalityChatbotConfig = {
    apiUrl: 'https://your-chatbot-server.com',  // Required: Your API URL
    apiKey: 'your-widget-api-key',          // Optional: Widget API key
    position: 'bottom-right',                // Optional: 'bottom-right' | 'bottom-left'
    themeColor: '#0066cc',                   // Optional: Primary color
    defaultLang: 'ar',                       // Optional: 'ar' | 'en'
    title: 'Municipality Assistant',          // Optional: Chat window title (English)
    titleAr: 'مساعد البلدية',                // Optional: Chat window title (Arabic)
  };
</script>
<script src="https://your-chatbot-server.com/widget/chatbot-widget.js"></script>

Configuration Options

Option Type Default Description
apiUrl string required The URL of your chatbot API server
apiKey string optional Widget API key for authentication
position string 'bottom-right' Widget position: 'bottom-right' or 'bottom-left'
themeColor string '#0066cc' Primary color for the widget
defaultLang string 'ar' Default language: 'ar' (Arabic) or 'en' (English)
title string 'Municipality Assistant' Chat window title (English)
titleAr string 'مساعد البلدية' Chat window title (Arabic)

JavaScript API

You can control the widget programmatically using the global MunicipalityChatbot object:

// Open the chat window
MunicipalityChatbot.open();

// Close the chat window
MunicipalityChatbot.close();

// Toggle the chat window
MunicipalityChatbot.toggle();

// Set language ('ar' or 'en')
MunicipalityChatbot.setLanguage('en');

// Send a message programmatically
MunicipalityChatbot.sendMessage('What are your working hours?');

Server Configuration

On the server side, configure the following environment variables:

# Allow widget requests from specific domains
Cors__WidgetAllowedOrigins="https://example.com,https://another-site.com"

# Or allow all domains (not recommended for production)
Cors__WidgetAllowedOrigins="*"

# Optional: Set a widget API key for authentication
WIDGET__API_KEY="your-secret-api-key"
