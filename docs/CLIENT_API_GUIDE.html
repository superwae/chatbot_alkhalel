Municipality Chatbot API - Integration Guide

Endpoint

POST http://egate.hebron-city.ps:8123/api/chat/public

Headers

Content-Type: application/json

Request Body

{
  "Message": "بدي اقدم شكوى",
  "Lang": "ar",
  "SessionId": null
}
FieldTypeRequiredDescription
MessagestringYesThe user's message
LangstringNo"ar" or "en" (auto-detected if omitted)
SessionIdstring (GUID)Nonull for first message. Must send back the sessionId from the response in all follow-up messages.
UserTokenstringNoCitizen auth token (for services requiring login, e.g. fees lookup)
IMPORTANT: Property names are PascalCase — use Message, not message.

Response

{
  "sessionId": "db380920-290b-46b6-96aa-2fc1ef66d69c",
  "route": "API",
  "answer": "the answer here",
  "citations": [],
  "followUpQuestion": null
}
FieldTypeDescription
sessionIdstring (GUID)Session ID — send this back in the next request
routestringWhich route was used (FAQ, API, RAG, GENERAL)
answerstringThe bot's answer (may contain URLs — render them as clickable links in your app)
citationsarraySource references (if any)
followUpQuestionstring or nullIf not null, show this instead of answer — the bot needs more info from the user

Multi-Turn Flow (Critical)

If you don't send SessionId back, every message is treated as a new conversation and multi-turn flows (complaints, follow-ups) will NOT work.
  1. First request → send "SessionId": null
  2. Get response → save the sessionId from the response
  3. All next requests → send that same SessionId back

Example: Full Complaint Flow

1 User starts complaint

// Request
{ "Message": "بدي اقدم شكوى", "Lang": "ar", "SessionId": null }

// Response
{
  "sessionId": "db380920-290b-46b6-96aa-2fc1ef66d69c",
  "answer": "",
  "followUpQuestion": "بالتأكيد! ما هي المشكلة التي تواجهها وأين موقعها؟"
}

2 User provides details (send sessionId back!)

// Request
{
  "Message": "مياه في شارع السلام",
  "Lang": "ar",
  "SessionId": "db380920-290b-46b6-96aa-2fc1ef66d69c"
}

// Response — bot asks for more details or shows confirmation

3 Continue until complaint is submitted

Always send the same SessionId in every request within the same conversation.

Display Logic

In your app, check the response and display accordingly: 
if (response.followUpQuestion != null)
    show followUpQuestion   // bot is asking the user for more info
else
    show answer             // bot's final answer

If answer contains URLs (e.g. https://www.hebron-city.ps), render them as clickable links.
